Responsive Layouts Without Bootstrap

This is a simple demonstration of organizing different elements without bootstrap so that when a browser is resized the elements adjust accordingly. The goal is obviously to be completely functional with as simple coding as possible. This means avoiding media queries when possible etc. Maybe this is common knowledge for most but as I'm growing and learning, I haven't found a comprehensive tutorial on the web with this stuff. SO..here's my two cents.

Working with Inline Block

Let's say you have a group of objects that are all similar in size and shape. A common scenario would be if you had multiple images or icons that you are setting side by side in a row. In our case, we'll use boxes. The easiest set-up is to set them in a parent container with text-align: center; to center them and then give each element a display:inline or inline-block. FYI, you do have to declare a width in order for this to work. Just keeping it real.

The cool thing this is that if you have an object of similar height but different widths, it still works! To keep it from overflowing on monitor sizes that are less than the width of the element put the max-width to 100% and it will cause it to resize at those resolutions.

If you want there to be two breaking points where a set of elements stay together a certain amount before breaking, which is common, then simply add two container elements to whatever elements you have in the rows and set that container element to display: inline-block.

Working with Flexbox

Ahh...imagine the day when all web browsers support flexbox. Oh, what a day that will be. Until then, my motto is if someone cared a ton about the way a website looked they'd probably be willing to upgrade their stinking browser. So, all in all, I think we spend a lot of time trying to impress people that don't even really care how wesbites look. Sorry, done with that rant. On to the example...The next scenario to tackle would be objects with different heights and widths. I've found that the best way to organize these is to use display: flex and the flex-wrap property to wrap. Obviously, there's a little extra work than our last solution. But, it still does a bang up job and solves the problem of encountering vertical allignment issues with objects different in size. A big kicker in all of this is that I've not found a way for flex-boxes to look good on mobile devices. They end up left aligning on me.

Working with Tables

I've tried having similar effects with display: table. It does fairly well in aligning elements of all shapes and sizes in rows. But, instead of adjusting rows upon monitor size change, it makes the elements squeeze proportionately. It may not look pretty here but there's certainly a chance it has its uses.

The only fix I've found is to set the inner elements to float left. Obviously, after the monitor is resized past a breaking point, instead of the shifting element ending up in the middle, it floats all the way to the left. And, it's not really necessary for the inner elements to be set at display: table-cell. And, you could do this if the container element were set to inline-block and floating the inner elements. So, all in all not a very good solution. Boo...
